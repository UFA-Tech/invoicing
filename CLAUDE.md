## Commands

```bash
npm run dev                          # Start dev server (Turbopack)
npm run build                        # Production build
npx tsc --noEmit                     # Type check only
npx prisma migrate dev --name <name> # Create and apply migration
npx prisma generate                  # Regenerate Prisma client after schema changes
npx prisma studio                    # Browse database
```

## Environment

Two env files are needed:

- `.env` — read by Prisma CLI (`DATABASE_URL` only)
- `.env.local` — read by Next.js (all variables including `DATABASE_URL`)

`prisma.config.ts` loads both files via dotenv with `.env.local` overriding `.env`. See `.env.local.example` for required keys.

## Critical Stack Differences

This project uses the **latest** package versions, which differ significantly from typical shadcn/ui and Prisma examples:

### shadcn/ui uses Base UI (not Radix UI)

`asChild` does **not** exist on any component. Use these patterns instead:

```tsx
// Buttons that navigate — use Link with buttonVariants
import { buttonVariants } from "@/components/ui/button";
<Link href="/foo" className={cn(buttonVariants({ variant: "outline", size: "sm" }))}>

// DropdownMenuTrigger — style directly, no child button needed
<DropdownMenuTrigger className="...your styles...">content</DropdownMenuTrigger>

// DropdownMenuItem navigation — use onClick + router.push, not asChild + Link
<DropdownMenuItem onClick={() => router.push("/foo")}>Go</DropdownMenuItem>

// PopoverTrigger — style directly with className
<PopoverTrigger className="flex h-8 w-full items-center ...">content</PopoverTrigger>
```

### Prisma 7

- `url` in `prisma/schema.prisma` datasource is invalid — connection is configured in `prisma.config.ts`
- Requires `@prisma/adapter-pg` (`PrismaPg`) in `lib/prisma.ts`; do not construct `PrismaClient` without it
- Prisma returns `Decimal` objects for all `Decimal` columns — always convert with `Number()` before passing to UI or PDF components
- After schema changes: run `npx prisma migrate dev` then `npx prisma generate`

### Zod 4

- Date schemas: `z.date({ error: "message" })` not `{ required_error: "message" }`
- ZodError: `.issues` not `.errors`

### Tailwind CSS 4

- No `tailwind.config.ts` — utilities are configured in `app/globals.css`

## Architecture

### Route Groups

```text
app/(auth)/          — Public routes (login). No auth check.
app/(dashboard)/     — All protected routes. Layout server-checks session via auth()
                       and redirects to /login if missing.
app/api/             — API routes. Every handler calls auth() and checks session.user.id.
```

### Data Security Pattern

Every DB query in API routes **must** filter by `userId`:

```ts
const invoice = await prisma.invoice.findFirst({
  where: { id, userId: session.user.id }, // never omit userId
});
```

### Invoice Number Format

`{prefix}{YYYY}-{MM}-{NNN}` e.g. `INV-2025-07-001`. Generated by `generateInvoiceNumber()` in `lib/utils.ts`. The prefix comes from `user.invoicePrefix`.

### Decimal Conversion

Prisma `Decimal` fields (`subtotal`, `taxRate`, `taxAmount`, `discount`, `total`, `quantity`, `unitPrice`, `amount`) must be cast with `Number()` in API route handlers before sending to client or to PDF:

```ts
subtotal: Number(invoice.subtotal),
items: invoice.items.map((i) => ({ ...i, quantity: Number(i.quantity), ... })),
```

### PDF Generation

`@react-pdf/renderer` runs server-side only. It is listed in `serverExternalPackages` in `next.config.ts`. The component receives already-normalized (plain JS numbers) data. Calling `renderToBuffer` requires a double `as any` cast:

```ts
const element = createElement(InvoicePDFTemplate as any, { invoice, business }) as any;
const pdfBuffer = await renderToBuffer(element);
return new NextResponse(new Uint8Array(pdfBuffer), { headers: { "Content-Type": "application/pdf" } });
```

PDF templates use built-in fonts only — **no external font URLs**. Bold text uses `fontFamily: "Helvetica-Bold"` rather than `fontWeight: 700` (which requires a registered custom font family to work).

### Form Handling Pattern

`InvoiceForm` is a large client component that owns the full form state via `react-hook-form` + Zod. Totals (`subtotal`, `taxAmount`, `total`) are **derived fields** stored in the form — they are recalculated on every qty/price/tax/discount change via `form.setValue`. The `InvoiceLineItems` component calls back into the parent form to trigger recalculation.

### `components/ui/form.tsx`

This file is **manually maintained** — it was not generated by shadcn CLI (the Base UI version of shadcn does not include a form component). Do not attempt to regenerate it via `npx shadcn add form`.

### Session Type

`types/next-auth.d.ts` augments `Session` to include `user.id`. The `auth()` call in server components and API routes returns this augmented type.

### Email + PDF Send Flow

`POST /api/invoices/[id]/send` fetches the PDF by making an internal HTTP call to `/api/invoices/[id]/pdf` (forwarding the session cookie), then passes the buffer to `sendInvoiceEmail()` in `lib/email.ts` as an attachment. It also updates `sentAt` and status to `SENT`.

## Key Files

| File | Purpose |
| ---- | ------- |
| `lib/prisma.ts` | Singleton Prisma client with `PrismaPg` adapter |
| `lib/auth.ts` | NextAuth config — Google OAuth, database sessions, `user.id` in session callback |
| `lib/utils.ts` | `formatCurrency`, `formatDate`, `generateInvoiceNumber`, `calculateInvoiceTotals` |
| `lib/email.ts` | Nodemailer transporter + `sendInvoiceEmail()` |
| `types/invoice.ts` | `InvoiceWithRelations`, `InvoiceFormData`, `BusinessProfile` interfaces |
| `prisma/schema.prisma` | Data model — no `url` in datasource (see `prisma.config.ts`) |
| `prisma.config.ts` | DB connection URL for Prisma CLI; loads both `.env` and `.env.local` |
| `next.config.ts` | `serverExternalPackages: ["@react-pdf/renderer"]`, `turbopack.root` |
